After installation, there is a PS Profile named very strangly (C:\Users\hoosi\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1) but open PowerShell and do:
1- C:\Users\hoosi> $PROFILE
2- Now just open a new PS windows and run "Start-SshAgent". This will save the ssh key. close the PS windows.
3- Now right click above "Microsoft.PowerShell_profile.ps1" and click edit to open it in PS ISE and add the following line:
Import-Module 'C:\source\posh-git\src\posh-git.psd1'
Start-SshAgent
save and close and reload the PS as normal user. This will atomatically get the id_rsa loaded so you don't have to enter password.
4- Open new PS as administrator to add path for git and run the following command:
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Program Files\Git\usr\bin", "Machine")

5- to put the content of id_rsa.pub (public key) into clipboard so we can paste it into GitHub web site.
Get-Content $env:USERPROFILE\.ssh\id_rsa.pub | clip

6- Finally, we can configure some global Git settings:

git config --global user.email "your@email.com"
git config --global user.name "Your Name"
git config --global push.default simple
git config --global core.ignorecase false

# Configure line endings for windows
git config --global core.autocrlf true

7- to test auth to GitHub, Open a USER PS windows: (Note & in the front of the ssh command)
C:\Users\hoosi> &"C:\Program Files\Git\usr\bin\ssh" -T git@github.com
Hi haghmiuni! You've successfully authenticated, but GitHub does not provide shell access.

Above command will create a clone of my-repo folder in "myprojects" folder

-----------------------------------------------------------------
Lets push now:
cd ~\myprojects\my-repo
create a sample file with notepad ( notepad mytest-file2)
git add -A
git commit -m "2nd file added"
git push origin master
-------------------------------------------------------------

Now I create a Branch from my master:
cd to cd ~\myprojects\my-repo  (master)
git checkout -b my-repo2       ( all files are exactly the same as mater for now- this also put you in repo2 folder)
notepad.exe mytest-file-branch  (create a new file in the branch)
git add -A
git commit -m "branch file added"
git push origin my-repo2           (will be added to brach on Github)
git branch
git status

---------------------------------------------------------------
What if you want to switch to master from branch!
git checkout master    (this would put you in master my-repo and notice that file are older so do )
git pull origin master
ls                      ( now you will see all files)
git status


----------------------------------------------------------------
Now you may want to accept the changes in Branch to Master:
Go to the Git hub and navigate to Branch (my-repo2)
And
click on "Merge Pull Request" button

All Branch files files are commited to Master!
